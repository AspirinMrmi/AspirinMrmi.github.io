Loaders

webpack只支持JS和JSON两种文件类型，对于那些不支持的文件类型就需要通过Loaders去进行转化处理。

常见的loader

名称 描述
babel-loader 转化es6以上的新语法
css-loader 支持.css文件的加载和解析
less-loader 将less文件转化成css
ts-loader 将TS转化为JS
file-loader 进行图片、字体等的打包
raw-loader 将文件以字符串的形式导入
thread-loader 多进程打包JS和CSS

用法

test 指定匹配规则

use 指定使用loader名称

plugins

用于bundle文件的优化，资源管理和环境变量的注入，作用于整个构建过程。可以说loaders做不到的事情都可以通过plugins去解决。

Mode

用来指定当前的构建环境是： production、development还是none。

设置Mode可以使用webpack内置的函数功能，默认为production。

选项 描述
development 设置process.env.NODE_ENV的值为development，开启NamedChunksPlugin和NameModulesPlugin
production 设置process.env.NODE_ENV的值为production，开启FlagDependencyUsagePlugin,FlagIncludedChunksPlugin,ModuleConcatenationPlugin,NoemitOnErrorsPlugin,OccurrenceOrderPlugin,SideEffectsFlagPlugin和TerserPluginl
none 不开启任何优化选项

webpack中的文件监听

文件监听

上面的学习过程中，非常费劲的一点就是每次更新代码我都需要重新去build，那么文件监听很有必要了解一下了。

文件监听就是在发现源码发生变化时，自动重新构建出新的输出文件。

webpack开启文件监听的方式：

  • 启动webpack命令时，带上--watch参数

  • 在配置webpack.config.js中设置watch: true

    我们尝试去修改package.json添加一个watch命令

这样我们每次执行的时候就去执行npm run watch，但是缺陷是每次修改代码我们都需要手动刷新浏览器。

原理分析

webpack会轮询（pool）判断文件的最后编辑时间是否发生变化。

某个文件发生变化，并不会立即告诉监听者，而是先缓存起来，缓存等待时间（aggregateTimeout）

文件指纹策略

打包后输出文件名的后缀

文件指纹的好处：

  • 版本管理
  • 浏览器缓存、页面加载更快

文件指纹分类：

  • Hash：和整个项目的构建相关，只要项目有文件做修改，整个项目的hahs值就会更改。
  • ChunkHash：和webpack打包生成的chunk有关，不同的entry会生成不同的chunkhash值。
  • ContentHash：根据文件内容来定义hash，文件内容不变，则contenthash不变。
